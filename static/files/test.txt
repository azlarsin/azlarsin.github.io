<h2 id="toc_1">通用说明部分</h2>

<h3 id="toc_2">1. URL</h3>

<ul>
<li><p>a. 说明</p>

<ol>
<li>url = 主域名 + 相对域名</li>
<li>api 中列出的域名均为相对域名，实际使用中可根据环境自由切换</li>
<li><strong>接受参数方式为 post 或 get 均可</strong></li>
<li><p>某些<strong>单一参数</strong>的方式，如 <a href="#toc_7">#1.1 获取验证码</a> 的域名可以为，无需指出具体参数名字 </p>

<div><pre class="line-numbers"><code class="language-none">http://api.test.com/v1/c/user/1 (Get)
http://api.test.com/v1/c/user/ (Post    {&quot;id&quot; = 1})</code></pre></div></li>
</ol></li>
<li><p>b. 测试环境</p>

<ol>
<li>主域名：http://123.56.154.187:8080/</li>
</ol></li>
<li><p>c. prod</p>

<ol>
<li>主域名：http://api.dogplanet.cn</li>
</ol></li>
</ul>

<h2 id="toc_6">1.    登录注册部分</h2>

<h3 id="toc_7">1.1  验证码通用接口</h3>

<ul>
<li><p>url:</p>

<div><pre class="line-numbers"><code class="language-none">/v1/c/user/get-code/</code></pre></div></li>
<li><p>params:   </p>

<div><pre class="line-numbers"><code class="language-none">1. phone    （Get or Post）
2. type 1   注册（默认）  2   找回密码</code></pre></div></li>
<li><p>response:</p>

<div><pre class="line-numbers"><code class="language-none">1. ret: 1001    已发送     0   发送失败（已注册）
2. msg: 成功（错误）提示</code></pre></div></li>
</ul>

<h3 id="toc_8">1.2  注册</h3>

<ul>
<li><p>url:</p>

<div><pre class="line-numbers"><code class="language-none">/v1/c/user/reg/</code></pre></div></li>
<li><p>params:</p>

<div><pre class="line-numbers"><code class="language-none">1. phone
2. code 验证码
3. name
4. password</code></pre></div></li>
<li><p>response:</p>

<div><pre class="line-numbers"><code class="language-none">1. ret: 1001    成功      0   失败
2. user =&gt; [
        id  =&gt; 1,

<h4 id="toc_36">6.2.2 购物车 -&gt; 生成订单</h4>

<li><p>说明:</p>

<li>生单过程中，有不能生成的订单（程序错误、数据错误等），会自动回滚，最后结果给出的订单号，即是成功生成的订单。（生单后，可去查询购物车数据，看看是否有添加失败的）</li>
<li>测试时，建议先 <a href="#toc_44">清空购物车</a> 从 <a href="#toc_25">5.1 购买服务页</a> 开始，找到有效的达人数据，然后 <a href="#toc_40">添加购物车</a> 再生成订单，这样可以确保数据的有效性，避免重复测试。</li>
<li>由于测试需要做好准备，故 sample 给出的例子不能保证时效性，请自行测试。</li>
</ol></li>
